The problem

Missions organizations run the world's hardest CRM on tools never built for it.

01 / Legacy

Closed systems age in the hardest places.

SiteStacker and its peers are aging, closed, and thin exactly where missions work is hardest.

02 / Generic

The record model is wrong.

Generic nonprofit CRMs collapse donor, missionary, and login into one record - the most expensive mistake there is.

03 / Unsafe

Person data can become a physical risk.

None treat a person record as what it can become in the wrong hands: a physical-safety risk.

Why this is different

The database is a targeting list.

In ~60 restricted-access countries, a leaked name, photo, or location is a physical-safety event - not a privacy incident.

Every record reveals religious belief by mere presence: special-category data by default.

Architecture

One system of record.

Asym Postgres owns all CRM and financial truth. Providers execute or store; they never own operational truth.

How we build

Four rules we don't break.

One source of truth

Every record type names its owner, its only write path, and its repair path. No record ships without an owner row.

Identity never collapses

A login is not a person, a donor, a missionary, or a Stripe customer - different records, linked, never merged.

Rules before ledger

The receipt and statement compliance brain was designed first; the money ledger is built to produce the facts those rules evaluate.

Classify before you build

Safety classification lands before custom fields or public pages can ever mint unclassifiable data. Retrofit is impossible.

Deep dive - Phase 10 / slug: sensitive-safety

The safety foundation, before anything can leak.

Four-tier classification

Public / internal / confidential / restricted as first-class metadata on fields, notes, and files.

Dual identity

Legal name vs public alias. Public surfaces read only a sanitized projection - never the real record.

RBAC + ABAC hybrid

Roles for coarse access; attribute predicates for record and field decisions. One policy engine.

Break-glass audit

Append-only sensitive-read logging plus emergency access with mandatory justification and review.

SiteStacker offers group permissions and security tags. This is a level beyond, and it must exist before custom fields or public missionary pages can ever create unclassifiable data.
